# SysWatch 🖥️

> Serveur de surveillance système en temps réel, accessible via TCP.  
> TP Intégral Rust — Génie Logiciel L4 | ENSPD 2025-2026

---

## Prérequis

- [Rust](https://rustup.rs) installé sur la machine
- Windows 10/11 avec PowerShell

Vérification :
```powershell
rustc --version
cargo --version
```

---

## Lancer le serveur

Ouvre un **terminal PowerShell** dans le dossier du projet et lance :

```powershell
cargo run
```

Tu dois voir s'afficher :

```
SysWatch démarrage...
[2026-04-21 09:13:50] Métriques initiales collectées avec succès
=== SysWatch — 2026-04-21 09:13:50 ===
CPU: 63.3% (8 cœurs)
MEM: 11604 Mo utilisés / 16244 Mo total (4639 Mo libres)
--- Top Processus ---
  [  3096] cmd.exe        CPU:102.9%  MEM: 2081 Mo
  ...
=====================
[2026-04-21 09:13:50] Serveur en écoute sur 0.0.0.0:7878
```

> ⚠️ **Ne ferme pas ce terminal** — le serveur doit rester actif.

## Se connecter et envoyer des commandes

Ouvre un **second terminal PowerShell** (clique sur `+` dans VS Code).

### Étape 1 — Connexion

Colle ce bloc en une seule fois :

```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8; $c = [System.Net.Sockets.TcpClient]::new("127.0.0.1", 7878); $r = [System.IO.StreamReader]::new($c.GetStream()); $w = [System.IO.StreamWriter]::new($c.GetStream()); $w.AutoFlush = $true; Write-Host $r.ReadLine(); Write-Host $r.ReadLine()
```

Tu dois voir :
```
=== SysWatch Server ===
Tapez 'help' pour la liste des commandes.
```

---

### Étape 2 — Envoyer des commandes

Copie-colle chaque ligne une par une :

**`help`** — liste des commandes
```powershell
$w.WriteLine("help"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()
```

**`cpu`** — utilisation CPU avec barre ASCII
```powershell
$w.WriteLine("cpu"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()
```

**`mem`** — utilisation RAM avec barre ASCII
```powershell
$w.WriteLine("mem"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()
```

**`ps`** — top 5 processus par CPU
```powershell
$w.WriteLine("ps"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()
```

**`all`** — rapport complet (CPU + RAM + processus)
```powershell
$w.WriteLine("all"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()
```

**`quit`** — fermer la connexion proprement
```powershell
$w.WriteLine("quit"); Start-Sleep 1; Write-Host $r.ReadLine(); $c.Close()
```

---

## Commandes disponibles

| Commande | Description |
|----------|-------------|
| `cpu`  | Utilisation CPU globale + barre ASCII |
| `mem`  | Utilisation RAM + barre ASCII + valeurs en Mo |
| `ps`   | Top 5 processus triés par consommation CPU |
| `all`  | Rapport complet : CPU + RAM + processus |
| `help` | Liste toutes les commandes disponibles |
| `quit` | Ferme la connexion avec le serveur |

## Journalisation

Toutes les connexions et commandes sont enregistrées automatiquement dans `syswatch.log`.

Pour lire le fichier de log :
```powershell
Get-Content .\syswatch.log
```

Exemple de contenu :
```
[2026-04-21 09:13:50] Serveur en écoute sur 0.0.0.0:7878
[2026-04-21 09:14:01] [+] Connexion de 127.0.0.1:52341
[2026-04-21 09:14:05] [127.0.0.1:52341] commande: 'cpu'
[2026-04-21 09:14:09] [127.0.0.1:52341] commande: 'quit'
[2026-04-21 09:14:09] [-] Déconnexion de 127.0.0.1:52341
```

## En cas de problème

**Erreur "connexion refusée"**
→ Le serveur n'est pas lancé. Va dans le terminal serveur et relance `cargo run`.

**`InvalidOperation: null-valued expression`**
→ La connexion est perdue. Relance le bloc de connexion de l'Étape 1.

**Terminal bloqué**
→ Ferme ce terminal (corbeille 🗑️ dans VS Code), ouvre-en un nouveau (`+`), puis relance le bloc de connexion.

> ⚠️ Ne jamais faire `Ctrl+C` dans le terminal serveur — cela arrête le serveur.  
> `Ctrl+C` uniquement dans le terminal client si le curseur est bloqué.
