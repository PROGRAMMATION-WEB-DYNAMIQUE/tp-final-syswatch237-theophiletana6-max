1. # PRÉREQUIS

- Rust installé sur la machine (https://rustup.rs)
- Windows 10/11 avec PowerShell
- VS Code (recommandé)

Vérifier que Rust est bien installé :

    rustc --version
    cargo --version

Si ces commandes ne fonctionnent pas, installe Rust sur :
https://rustup.rs puis ferme et réouvre PowerShell.

================================================================ 3. LANCER LE SERVEUR
================================================================

IMPORTANT : Ce terminal doit rester ouvert pendant toute
la durée d'utilisation du serveur.

Ouvre un terminal PowerShell dans le dossier du projet :

    cd C:\tp-final-syswatch237-theophiletana6-max\syswatch

Lance le serveur :

    cargo run

Tu dois voir s'afficher :

    SysWatch démarrage...
    [2026-04-21 09:13:50] Métriques initiales collectées avec succès
    === SysWatch — 2026-04-21 09:13:50 ===
    CPU: 63.3% (8 cœurs)
    MEM: 11604 Mo utilisés / 16244 Mo total (4639 Mo libres)
    --- Top Processus ---
      [  3096] cmd.exe        CPU:102.9%  MEM: 2081 Mo
      [ 27724] Code.exe       CPU: 87.5%  MEM:  101 Mo
      ...
    =====================
    [2026-04-21 09:13:50] Serveur en écoute sur 0.0.0.0:7878

Le serveur rafraîchit les métriques toutes les 5 secondes
automatiquement.

⚠️ NE PAS faire Ctrl+C dans ce terminal — cela arrêterait
le serveur.

================================================================ 4. SE CONNECTER DEPUIS LE MÊME ORDINATEUR
================================================================

Ouvre un SECOND terminal PowerShell (clique sur "+" dans VS Code).

---- ÉTAPE 1 : Connexion ----

Colle ce bloc entier en une seule fois et appuie sur Entrée :

    [Console]::OutputEncoding = [System.Text.Encoding]::UTF8; $c = [System.Net.Sockets.TcpClient]::new("127.0.0.1", 7878); $r = [System.IO.StreamReader]::new($c.GetStream()); $w = [System.IO.StreamWriter]::new($c.GetStream()); $w.AutoFlush = $true; Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Tu dois voir :

    === SysWatch Server ===
    Tapez 'help' pour la liste des commandes.

---- ÉTAPE 2 : Envoyer des commandes ----

Copie-colle chaque ligne une par une :

Commande HELP :
$w.WriteLine("help"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Commande CPU :
$w.WriteLine("cpu"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Commande MEM :
$w.WriteLine("mem"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Commande PS :
$w.WriteLine("ps"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Commande ALL :
$w.WriteLine("all"); Start-Sleep 1; Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Commande QUIT (fermer la connexion) :
$w.WriteLine("quit"); Start-Sleep 1; Write-Host $r.ReadLine(); $c.Close()

================================================================ 5. SE CONNECTER DEPUIS UN AUTRE ORDINATEUR
================================================================

Les deux machines doivent être sur le MÊME réseau local
(même Wi-Fi ou même câble réseau).

---- ÉTAPE 1 : Trouver l'adresse IP du serveur ----

Sur la machine qui fait tourner le serveur, ouvre PowerShell
et tape :

    ipconfig

Cherche la ligne "Adresse IPv4", par exemple :

    Adresse IPv4. . . . . . . . . . . : 192.168.1.45

Note cette adresse IP — tu en auras besoin sur l'autre machine.

---- ÉTAPE 2 : Ouvrir le port 7878 dans le pare-feu ----

Sur la machine SERVEUR, ouvre PowerShell en tant
qu'ADMINISTRATEUR et exécute :

    New-NetFirewallRule -DisplayName "SysWatch" -Direction Inbound -Protocol TCP -LocalPort 7878 -Action Allow

---- ÉTAPE 3 : Connexion depuis l'autre machine ----

Sur l'AUTRE ordinateur, ouvre PowerShell et remplace
192.168.1.45 par l'IP réelle du serveur :

    [Console]::OutputEncoding = [System.Text.Encoding]::UTF8; $c = [System.Net.Sockets.TcpClient]::new("192.168.1.45", 7878); $r = [System.IO.StreamReader]::new($c.GetStream()); $w = [System.IO.StreamWriter]::new($c.GetStream()); $w.AutoFlush = $true; Write-Host $r.ReadLine(); Write-Host $r.ReadLine()

Ensuite, les commandes sont exactement les mêmes que dans
la section 4 (CPU, MEM, PS, ALL, HELP, QUIT).

================================================================ 7. LIRE LE FICHIER DE LOG
================================================================

Le serveur enregistre automatiquement toutes les connexions
et commandes dans le fichier syswatch.log.

Pour lire ce fichier :

    Get-Content .\syswatch.log

Exemple de contenu :

================================================================ 8. EN CAS DE PROBLÈME
================================================================

PROBLÈME : "connexion refusée" / "Aucune connexion n'a pu être établie"
SOLUTION : Le serveur n'est pas lancé ou s'est arrêté.
Va dans le terminal serveur et relance : cargo run

---

PROBLÈME : "InvalidOperation: null-valued expression"
SOLUTION : La connexion TCP est perdue.
Relance le bloc de connexion complet de l'Étape 1
(section 4).

---

PROBLÈME : Le terminal client est bloqué (curseur fixe)
SOLUTION : 1. Appuie sur Ctrl+C dans le terminal CLIENT
(jamais dans le terminal serveur) 2. Ferme ce terminal (corbeille dans VS Code) 3. Ouvre un nouveau terminal (+) 4. Relance le bloc de connexion de l'Étape 1

---

PROBLÈME : Les barres ASCII █░ s'affichent mal
SOLUTION : Ajoute cette ligne au début de chaque session :
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
(déjà inclus dans le bloc de connexion)

---

PROBLÈME : "cargo : commande introuvable"
SOLUTION : Rust n'est pas dans le PATH.
Ferme et réouvre PowerShell après installation,
ou exécute :
$env:PATH += ";$env:USERPROFILE\.cargo\bin"

---

⚠️ RÈGLE D'OR : Ne jamais faire Ctrl+C dans le terminal
serveur. Uniquement dans le terminal client si bloqué.
